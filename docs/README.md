# docs

项目文档中心，确保“设计可追溯、决策可复盘、上线可运维”。

## 子目录说明
- `architecture/`：系统架构与模块设计
- `plans/`：阶段计划与任务拆解
- `adr/`：架构决策记录（Architecture Decision Record）
- `api/`：接口与协议文档
- `runbooks/`：运维与故障处理手册

## 快速入口
- 本地启动顺序：`runbooks/local/web_mvp_local_startup.md`
- 冒烟清单：`runbooks/release/smoke_checklist.md`

## 维护规则
- 任何重大技术决策都应新增 ADR。
- 文档优先写“为什么”和“边界”，不只写“怎么做”。
