# 2026-02-09 实施路线图（Web/PWA 优先）

## Phase 0：资产与管线打底
1. MMD 资产工程化（模型、动作、表情、口型可用性检查）。
2. 人物卡解析器（SillyTavern -> 系统提示词模板）。
3. GPT-SoVITS 服务封装（文本+情绪 -> 音频输出）。

## Phase 1：MVP（网页）
1. 文本对话链路。
2. 按住说话语音链路（ASR -> LLM -> TTS）。
3. Web 3D 角色 Idle + Speak + 口型同步。
4. 关系值与基础事件系统。

## Phase 2：沉浸升级
1. VAD 免按键通话。
2. 情绪语音与动作联动。
3. 触摸互动、换装、房间、回忆相册。

## 验收基线
- 人格一致性稳定。
- 首句语音响应延迟可控。
- 隐私与数据删除能力可用。
- Web 端移动浏览器可用（PWA 可安装）。

## P0-P4 总体开发流程（当前版本）

统一工作流（每个 Pi 都按同一套路走，避免返工）：
1. 构思方案：明确范围、风险红线、验收标准、数据与合规边界。
2. 提请审核：把方案落到 `docs/plans/YYYY-MM-DD-*.md`，由你确认取舍。
3. 分解任务：拆成“可交付物 + 验收口径 + 执行者（强视觉AI/强逻辑AI）”。
4. 执行实现：按分工并行推进，任何跨域任务必须拆成两条（视觉/逻辑分离）。
5. 回归验收：功能回归优先；性能仅做主观体验验证（不做量化基准）。
6. 收尾沉淀：更新 registry/manifest/提示词/操作文档，记录风险与回滚点。

分工原则：
- **强视觉AI**：多模态/图片/视频/画面理解，美术，UI/UX，前端视觉表现，平面设计，动效与氛围营造。
- **强逻辑AI**：强逻辑，后端，算法，状态机与数据结构，工程化重构，资源释放与稳定性保障。

里程碑定义（P0-P4）：
- **P0（资产与合规）**：Model/Motion Registry、命名规范、清单化（manifest）、忽略策略（不入库资产只入库登记）。
- **P1（MVP 对话闭环）**：文本对话 + 按键语音链路 + 基础 3D 表现 + 隐私开关。
- **P2（MMD 集成稳态）**：MMD 渲染/动作状态机/口型与材质修复、语音能力门控与基础回归。
- **P3（沉浸升级）**：VAD、触摸互动、表情系统、房间与相册、换装（模型切换）与体验打磨。
- **P4（商业化与安全）**：VIP 后端鉴权中间件、订阅/过期、支付与充值页、安全与日志脱敏加固。
