## 清单

| 编号 | 检查项 | 操作 | 通过标准 |
| --- | --- | --- | --- |
| SMK-01 | 配置校验 | `pwsh .\scripts\validation\validate_configs.ps1` | 脚本 0 退出，输出“配置校验通过” |
| SMK-02 | 服务端口 | 检查 `50000/9880/8000/3000` 端口连通性 | 四个端口均可连通 |
| SMK-03 | Server 健康检查 | `GET http://127.0.0.1:8000/healthz` | 返回 200 且包含 `ok` |
| SMK-04 | Web 可访问 | 打开 `http://127.0.0.1:3000` | 页面可加载，无白屏 |
| SMK-05 | TTS 代理链路 | 触发一次语音合成（使用可用 `ref_audio_path`） | Server 返回音频，前端可播放 |
| SMK-06 | 语音链路错误可见性 | 人为停止 SenseVoice 后触发语音输入 | UI 有明确失败提示，且可恢复 |
| SMK-07 | 3D 状态切换 | 触发一次“静止→说话→静止” | 状态切换可观察，无卡死 |
| SMK-08 | 隐私清理入口 | 执行一次“清理记忆/会话数据”操作 | 清理成功提示明确，可继续对话 |

## 结果判定
- 阻断项：`SMK-01`~`SMK-05` 任一失败即判定 **不通过**。
- 非阻断项：`SMK-06`~`SMK-08` 失败需登记风险与修复计划。
